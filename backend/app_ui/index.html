配置 OpenClaw 或 CLI 助手

设置环境变量后，所有发出的对话请求都会自动经过记忆代理层，对话内容被自动记录，相关历史被自动注入。

# 设置代理地址（所有对话会自动记忆化） export OPENAI_BASE_URL="http://localhost:8003/v1" export OPENAI_API_KEY="mos_xxxx" # 启动 OpenClaw — 无需任何其他配置 openclaw # 原理: 当 OpenClaw 发出任意 /v1/chat/completions 请求时： # 1. Memory OS 自动检索您过去相关的操作记录和知识 # 2. 将其压缩注入到 system prompt（节省 Token） # 3. 对话结束后自动将新内容存入记忆库

配置桌面智能体 (Claude Desktop / Hermes / 爱马仕)

在客户端"高级设置 → 自定义 API 提供商"中填写以下配置，让桌面端应用共享同一个记忆网络。

# Claude Desktop / Hermes 配置填写位置： # 设置 → 高级 → 自定义 Model Provider Base URL (API Endpoint): http://localhost:8003/v1 API Key: mos_xxxx # 重要: 模型名称填任意值（如 gpt-4o）， # Memory OS 会路由到您在管理端配置的真实模型。 # 所有对话自动双向记忆化（检索 + 存储）。

配置 AI 编程助手 (Cursor / Codex / Antigravity)

IDE 插件接入后，助手能自动联想您过去的架构规范、报错处理记录，无需重复说明项目背景。

// Cursor / Codex settings.json 配置 { "openai.apiKey": "mos_xxxx", "openai.baseUrl": "http://localhost:8003/v1", // Antigravity (Gemini) 插件配置示例: "antigravity.provider": "custom", "antigravity.endpoint": "http://localhost:8003/v1", "antigravity.apiKey": "mos_xxxx" } // 效果: 每次 AI 补全请求都经过代理层： // 1. 自动检索您上传的文档/规范/历史报错 // 2. 将相关内容注入 context（按需，不超 500 tokens） // 3. 本次代码问答自动记录为新记忆

Python 代理接入 (自动记忆化)

直接使用 openai 库，只需改两行，即可让所有对话自动经过记忆库（检索 + 存储），无需任何额外逻辑。

✅ 使用代理模式：系统自动完成「记忆检索注入」和「对话存储」，Token 消耗大幅降低
from openai import OpenAI # 只需修改这两行，其余代码不动 client = OpenAI( base_url="http://localhost:8003/v1", # 指向 Memory OS 代理 api_key="mos_xxxx" # 您的专属 Token ) # 像普通 OpenAI SDK 一样使用 —— 记忆自动运作 response = client.chat.completions.create( model="qwen-turbo", # 任意模型名，由后台路由 messages=[ {"role": "user", "content": "帮我回顾一下上周关于 PostgreSQL 优化的讨论"} ] ) # Memory OS 在背后做的事： # 1. 接收到您的 "PostgreSQL 优化" 请求 # 2. 语义检索知识库，找到相关记忆片段（Top-3） # 3. 自动注入 system_prompt（仅 ~300-500 tokens） # 4. 将完整 messages 转发给真实大模型 # 5. 将本次对话自动存为新记忆 print(response.choices[0].message.content)

直接调用记忆 API（高级用法）

如需手动控制检索逻辑，可直接调用原生 Memory API。

import httpx BASE = "http://localhost:8003" HEADERS = {"Authorization": "Bearer mos_xxxx"} # 手动存入记忆 httpx.post(f"{BASE}/memory/store", headers=HEADERS, json={ "title": "PostgreSQL 分区表优化方案", "content": "使用 BRIN 索引替代 B-tree 可降低 90% 的索引体积...", "source_type": "human" }) # 手动语义检索（返回最相关 Top-5） results = httpx.post(f"{BASE}/memory/search", headers=HEADERS, json={ "query": "数据库索引优化方法", "top_k": 5 }).json() for r in results: m = r.get("memory", r) print(f"[{m['lifecycle_stage']}] {m['title']}: {m['content'][:100]}...")
